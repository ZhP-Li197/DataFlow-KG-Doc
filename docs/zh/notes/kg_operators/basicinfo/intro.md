## DataFlow-KG 算子体系说明

`DataFlow-KG` 面向不同类型知识图谱的构建、处理、优化与评估需求，设计了一套结构清晰、可扩展、易复用的算子体系。整体上，`DataFlow-KG` 的算子组织方式、设计原则与命名规范如下。

### 1. 按图谱类型组织算子

`DataFlow-KG` 按照不同类型的知识图谱对算子进行组织。每一种图谱类型对应一个独立文件夹，文件夹内部再按照算子的功能阶段划分为以下四类：

* `generate`：用于图谱生成与构建
* `filter`：用于数据筛选与质量过滤
* `refine`：用于图谱优化、补全与规范化处理
* `eval`：用于图谱质量评估与效果验证

这种组织方式使不同图谱类型下的算子结构保持一致，便于开发、维护与后续扩展。

目前，`DataFlow-KG` 支持的图谱类型包括但不限于：

* 通用图谱（General KG）
* 常识图谱（Commonsense KG）
* 时序图谱（Temporal KG）
* 多模态图谱（Multimodal KG）
* 超关系图谱（Hyper-relational KG）

### 2. 算子设计原则：高复用、低重复

`DataFlow-KG` 的算子设计遵循**高复用、低重复**的原则。

其中，**通用图谱**——对应general_kg文件夹——主要表示最常见的知识图谱结构，例如：

* 实体–关系–实体三元组
* 实体–属性–属性值三元组

通用图谱对应的算子体系构成了整个 `DataFlow-KG` 的**基础能力层**。其中，`filter` 和 `refine` 类算子具备较强的通用性，能够适配大多数其他类型的图谱任务。因此，在设计常识图谱、时序图谱、多模态图谱、超关系图谱等其他类型图谱时，我们尽可能复用通用图谱中的已有算子，从而减少重复开发成本，并提升整体框架的一致性与可维护性。

### 3. 算子命名规则

`DataFlow-KG` 中的算子文件统一采用如下命名格式：

```text
[图谱类型]_[对象]_[具体功能]
```

例如：

```text
kg_entity_extract
tkg_tuple_filter
mmkg_qa_generate
hrkg_subgraph_refine
```

该命名方式能够直接反映算子的适用图谱类型、处理对象与核心功能，便于快速理解与检索。

#### 3.1 图谱类型缩写

| 图谱类型  | 缩写     | 说明                 |
| ----- | ------ | ------------------ |
| 通用图谱  | `kg`   | 面向通用实体关系与属性表达的基础图谱 |
| 常识图谱  | `ckg`  | 面向常识知识表示与推理的图谱     |
| 时序图谱  | `tkg`  | 面向带时间信息的动态图谱       |
| 多模态图谱 | `mmkg` | 面向文本、图像等多模态信息融合的图谱 |
| 超关系图谱 | `hrkg` | 面向高阶关系与复杂语义结构的图谱   |

#### 3.2 对象类型说明

| 对象         | 含义   | 说明               |
| ---------- | ---- | ---------------- |
| `entity`   | 实体   | 针对实体本身进行处理       |
| `rel`      | 关系元组 | 表示实体–关系–实体类型的数据  |
| `attri`    | 属性元组 | 表示实体–属性–属性值类型的数据 |
| `triple`   | 三元组  | 泛指三元结构数据         |
| `tuple`    | 多元组  | 表示四元组及以上更高元结构    |
| `qa`       | 问答   | 面向图谱问答构建、处理或评估   |
| `subgraph` | 子图   | 面向局部图结构处理        |
| `path`     | 路径   | 面向图路径挖掘、推理或分析    |


### 4. 领域图谱算子设计

除了面向图谱类型的通用算子体系之外，`DataFlow-KG` 还针对常见应用场景设计了**领域图谱**算子，对应domain_kg文件夹。

与类型图谱保持一致，领域图谱中的算子同样按照以下四类进行组织：

* `generate`
* `filter`
* `refine`
* `eval`

领域图谱更关注特定场景下的实际应用需求，强调任务导向与应用适配。我们也充分认识到，垂直领域中的应用场景极为丰富，现有算子体系仍有很大的扩展空间。因此，我们欢迎社区开发者围绕更多领域场景贡献新的应用算子，共同完善 `DataFlow-KG` 的生态能力。

### 5. 设计理念总结

总体而言，`DataFlow-KG` 的算子体系具有以下特点：

* **结构清晰**：按照图谱类型与功能阶段进行分层组织
* **规范统一**：采用统一的命名规则与目录设计
* **高度复用**：以通用图谱算子作为基础能力层，尽可能服务于多种图谱类型
* **易于扩展**：支持面向新图谱类型与新领域场景的持续扩展
* **面向应用**：兼顾通用能力建设与领域应用落地
