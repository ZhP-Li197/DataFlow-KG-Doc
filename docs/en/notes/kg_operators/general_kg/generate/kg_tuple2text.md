## 📚 Overview
`KGTupleTextGeneration` converts KG triples or tuple lists into natural language text. It reads the structured triples in each row, calls an LLM to generate a descriptive paragraph, and writes the text result back to the DataFrame.

The operator first tries to use the explicitly specified input column. If that is not available, it can automatically fall back to either `triple` or `tuple` in the DataFrame. The model output is not parsed as JSON; it is stored directly as the `description` text.

## ✒️ `__init__` Function
```python
def __init__(
    self,
    llm_serving: LLMServingABC,
    seed: int = 0,
    lang: str = "en",
):
    ...
```

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `llm_serving` | `LLMServingABC` | - | LLM backend used for text generation. |
| `seed` | `int` | `0` | Seed for the internal random generator. |
| `lang` | `str` | `"en"` | Prompt language. |

## 💡 run Function
```python
def run(
    self,
    storage: DataFlowStorage = None,
    input_key_meta: str = "triple",
    output_key_meta: str = "description"
):
    ...
```

`run` first sets the input and output column names, reads the DataFrame, and then calls `_validate_dataframe()`. That validator prefers the explicitly provided input column, otherwise it auto-selects either `triple` or `tuple`. Next, `process_batch()` builds prompts row by row, calls the LLM, and stores the stripped model output directly as `description`. The results are then written to `output_key_meta`.

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `storage` | `DataFlowStorage` | `None` | Storage object for the DataFrame. |
| `input_key_meta` | `str` | `"triple"` | Input column containing triples. |
| `output_key_meta` | `str` | `"description"` | Output column for natural-language descriptions. |

## 🤖 Example Usage
```python
from dataflow.operators.general_kg.generate.kg_tuple2text import (
    KGTupleTextGeneration,
)

operator = KGTupleTextGeneration(
    llm_serving=llm_serving,
    lang="en",
)

operator.run(
    storage=storage,
    input_key_meta="triple",
    output_key_meta="description",
)
```

Default input and output format:

| Field | Type | Description |
| :-- | :-- | :-- |
| `triple` / `tuple` | `List[str]` | Input triples or tuples. |
| `description` | `str` | Natural-language paragraph generated by the model. |

Example input:

```json
[
  {
    "triple": [
      "<subj> Ada Lovelace <obj> Charles Babbage <rel> collaborated_with",
      "<subj> Ada Lovelace <obj> analytical engine <rel> wrote_about"
    ]
  }
]
```

Example output:

```json
[
  {
    "description": "Ada Lovelace collaborated with Charles Babbage and wrote about the analytical engine."
  }
]
```
