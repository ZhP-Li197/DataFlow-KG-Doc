## 📚 Overview
`KGQAConciseFilter` is a filtering operator that filters QA pairs based on conciseness scores.
It reads the list of QA pairs and their corresponding conciseness score list from each row, aligns them by position, retains QA pairs whose scores fall within the specified range, and writes the filtered results back to the DataFrame.

Key characteristics of this operator:

- Does **not** require `LLMServingABC` — this is a pure rule-based filtering operator with no dependency on a large language model
- QA pairs and scores are aligned one-to-one by position; each pair is evaluated individually for retention
- Supports writing filtered results back in-place to the input column via `merge_to_input`, or writing to a new column
- QA pairs with a score of `None` are filtered out
- Typically used with the output column of `KGQAConcisenessEvaluator`

```python
def __init__(self, merge_to_input: bool = False):
    ...
```

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `merge_to_input` | `bool` | `False` | If `True`, filtered results are written back to the `input_key` column and `output_key` is ignored. If `False`, results are written to the column specified by `output_key`. |

## 💡 `run` Parameters
```python
def run(
    self,
    storage: DataFlowStorage,
    input_key: str = "QA_pairs",
    score_key: str = "conciseness_scores",
    output_key: str = "filtered_QA_pairs",
    min_score: float = 0.95,
    max_score: float = 1.0,
):
    ...
```

`run` reads the DataFrame from `storage`, validates that both `input_key` and `score_key` columns exist, then aligns the QA pair list with the score list by position for each row, retaining pairs that satisfy `min_score ≤ score ≤ max_score`. If the input is not a list type, an empty list is output for that row. Based on `merge_to_input`, results are written to either the `input_key` or `output_key` column.

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `storage` | `DataFlowStorage` | - | Dataflow storage object. The operator reads the `dataframe` from it and writes the filtered results back. |
| `input_key` | `str` | `"QA_pairs"` | Input QA pair column name. Each cell should be in `List[Dict]` format. |
| `score_key` | `str` | `"conciseness_scores"` | Input conciseness score column name, typically the output column of `KGQAConcisenessEvaluator`. Each cell should be a `List[float]` of the same length as `input_key`. |
| `output_key` | `str` | `"filtered_QA_pairs"` | Output filtered results column name (effective when `merge_to_input=False`). |
| `min_score` | `float` | `0.95` | Minimum score threshold (inclusive) for retaining a QA pair. |
| `max_score` | `float` | `1.0` | Maximum score threshold (inclusive) for retaining a QA pair. |

## 🤖 Example Usage
```python
from dataflow.operators.general_kg.filter.kg_qa_concise_filtering import (
    KGQAConciseFilter,
)

operator = KGQAConciseFilter(merge_to_input=False)
operator.run(
    storage=storage,
    input_key="QA_pairs",
    score_key="conciseness_scores",
    output_key="filtered_QA_pairs",
    min_score=0.95,
    max_score=1.0,
)
```

---

#### Default Output Format
| Field | Type | Description |
| :-- | :-- | :-- |
| `QA_pairs` | `List[Dict]` | Original input QA pair list. |
| `conciseness_scores` | `List[float]` | Conciseness scores for each QA pair, generated by the upstream evaluation operator. |
| `filtered_QA_pairs` | `List[Dict]` | Filtered QA pair list retaining only entries whose scores fall within `[min_score, max_score]`. |
