## 📚 Overview
`KGTripleStrengthFilter` is a filtering operator that filters knowledge graph triples based on strength scores.
It reads the list of triples and their corresponding strength score list from each row, aligns them by position, retains triples whose scores fall within the specified range, and writes the filtered results back to the DataFrame.

Key characteristics of this operator:

- Does **not** require `LLMServingABC` — this is a pure rule-based filtering operator with no dependency on a large language model
- Triples and scores are aligned one-to-one by position; each triple is evaluated individually for retention
- Supports writing filtered results back in-place to the input column via `merge_to_input`, or writing to a new column
- Triples with a score of `None` are filtered out
- Typically used with the output column of `KGRelationStrengthScoring`

## ✒️ `__init__` Parameters
```python
def __init__(self, merge_to_input: bool = False):
    ...
```

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `merge_to_input` | `bool` | `False` | If `True`, filtered results are written back to the `triple_key` column and `output_key` is ignored. If `False`, results are written to the column specified by `output_key`. |

---

## 💡 `run` Parameters
```python
def run(
    self,
    storage: DataFlowStorage,
    triple_key: str = "triple",
    score_key: str = "triple_strength_score",
    output_key: str = "filtered_triple",
    min_score: float = 0.5,
    max_score: float = 1.0,
):
    ...
```

`run` reads the DataFrame from `storage`, validates that both `triple_key` and `score_key` columns exist, then aligns the triple list with the score list by position for each row, retaining triples that satisfy `min_score ≤ score ≤ max_score`. If the input is not a list type, an empty list is output for that row. Based on `merge_to_input`, results are written to either the `triple_key` or `output_key` column.

> Note: Unlike QA filtering operators, the first input parameter of this operator is named `triple_key` (not `input_key`). Pay attention to the parameter name when calling.

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `storage` | `DataFlowStorage` | - | Dataflow storage object. The operator reads the `dataframe` from it and writes the filtered results back. |
| `triple_key` | `str` | `"triple"` | Input triple column name. Each cell should be in `List[str]` format. |
| `score_key` | `str` | `"triple_strength_score"` | Input strength score column name, typically the output column of `KGRelationStrengthScoring`. Each cell should be a `List[float]` of the same length as `triple_key`. |
| `output_key` | `str` | `"filtered_triple"` | Output filtered results column name (effective when `merge_to_input=False`). |
| `min_score` | `float` | `0.5` | Minimum score threshold (inclusive) for retaining a triple. |
| `max_score` | `float` | `1.0` | Maximum score threshold (inclusive) for retaining a triple. |

## 🤖 Example Usage
```python
from dataflow.operators.general_kg.filter.kg_rel_triple_strength_filtering import (
    KGTripleStrengthFilter,
)

operator = KGTripleStrengthFilter(merge_to_input=False)
operator.run(
    storage=storage,
    triple_key="triple",
    score_key="triple_strength_score",
    output_key="filtered_triple",
    min_score=0.5,
    max_score=1.0,
)
```

---

#### Default Output Format
| Field | Type | Description |
| :-- | :-- | :-- |
| `triple` | `List[str]` | Original input triple list. |
| `triple_strength_score` | `List[float]` / `List[float\|None]` | Strength scores for each triple, generated by the upstream evaluation operator. |
| `filtered_triple` | `List[str]` | Filtered triple list retaining only entries whose scores fall within `[min_score, max_score]`. |
